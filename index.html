Step 02

외부 공유없이 로컬 PC에서만 쓰는 HTML, 안전한가요?

견적서 양식, 내부 업무 도구 등 로컬에서만 여는 HTML도 간단한 점검이 필요합니다.

🔍 2가지만 확인하세요
#점검 항목확인 방법
1알 수 없는 외부 URL로 데이터를 보내고 있지 않은지Ctrl+F → http 검색, 모르는 주소 제거
2비밀번호나 API 키가 코드에 그대로 적혀 있지 않은지Ctrl+F → password, key 검색
Key Point

로컬 HTML이라도 외부에서 받은 템플릿이나 AI로 생성한 코드에는 외부 전송 코드가 숨어 있을 수 있습니다. http로 시작하는 주소가 있는지 한 번만 확인해 주세요.

Step 03

외부에 올리는 서비스, 이것만은 꼭!

GitHub Pages, 웹호스팅 등 외부 서버에 올리는 HTML/JS 서비스의 필수 보안 점검 항목입니다.

⚠ 배포 전 필수 6가지 점검
AI로 간편하게 점검하기

#점검 항목미점검 시 위험확인 방법
1비밀번호 해시(SHA-256) 저장DB 유출 시 PW 전부 노출sha256 검색
2Firebase Security Rules 설정API 키로 누구든 데이터 삭제 가능Firebase 콘솔 > 규칙
3민감 API 키 코드 노출결제 키 노출 시 무단 결제key, secret 검색
4입력값 이스케이프 (XSS 방지)악성 코드가 사용자 화면에서 실행textContent 확인
5Firebase 컬렉션 접두사"users" 등 이름은 공격 대상서비스명-기능 형태
6F12 개발자도구 우회 방지콘솔에서 인증 우회 가능인증 체크 코드
🔒 GitHub 소스코드 공개 문제
현재 상태
GitHub 무료(Public)로 배포 중이라 소스코드가 누구에게나 공개되어 있습니다.

Step 07

이 메일, 진짜일까요?

피싱 메일을 구별하는 방법과, 받았을 때 어떻게 해야 하는지 안내합니다.

특히 메타(Facebook) 관련 메일을 주의하세요
'계정 제한', '보안 문제', '광고 중단' 내용의 메일은 피싱 가능성이 매우 높습니다. 직접 대응하지 말고 경영관리팀에 먼저 공유하세요.
⚠ 이런 메일은 의심하세요
  • "계정이 제한됩니다", "즉시 확인하세요" 등 급한 문구
  • "광고가 중단되었습니다" 등 업무 사칭
  • 공식이 아닌 비슷한 발신자 주소
    예: @faceb00k-support.com
  • "거래명세서", "세금계산서" 등 업종 타겟형
  • 첨부파일이 .exe .zip .scr 등 실행 파일
✓ 이렇게 대응하세요
  1. 메일 속 링크 클릭 금지, 첨부파일도 열지 마세요
  2. 경영관리팀에 먼저 전달 (잔디 또는 유선)
  3. 확인 필요 시 메일 링크가 아닌 사이트에 직접 접속
  4. 이미 클릭했다면 즉시 경영관리팀 연락
  5. 정보를 입력했다면 비밀번호 즉시 변경
간단 판별법

진짜 서비스는 메일로 비밀번호나 개인정보를 요구하지 않습니다.
"지금 당장 안 하면 큰일난다"는 식일수록 피싱 확률이 높습니다.

Step 09

AI 도구, 뭐가 되고 뭐가 안 되나요?

같은 AI라도 작동 방식이 다릅니다. 안전한 것과 위험한 것을 구분해 드립니다.

✓ Claude / ChatGPT (사용 가능)
  • 내가 입력한 내용만 AI가 처리
  • AI가 PC 파일을 마음대로 못 열어봄
  • 뭘 보내는지 내가 통제 가능
⚠ 오픈클로 등 에이전트형 AI (금지)
  • PC 전체 접근 권한 보유
  • 파일 열기/수정/삭제/전송 가능
